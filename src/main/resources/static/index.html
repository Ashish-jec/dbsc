DBSC API Test

Device Bound Session Credentials — test registration, refresh, and protected API

1. Start session

1 GET /dbsc/session/start → read Secure-Session-Registration (challenge + path)


  


  

    
2. Register (DBSC proof)

    
2 Generate ES256 key, sign JWT (jti = challenge), POST to registration with Secure-Session-Response

    
    

    

  


  

    
3. Call protected API

    
3 GET /api/secure with session cookie. Cookie expires in 30s; if you get 401, the page will auto-refresh (only works here — this device has the key) and retry.

    
    

    

  


  

    
Debug

    
After registering, click to see what the server receives (same cookies as other requests).

    
    

    

  


  

    
4. Refresh (optional)

    
4 POST /dbsc/refresh with X-Dbsc-Session-Id + Secure-Session-Response (signed with cached challenge). Note: JS cannot set Sec-Secure-Session-Id; server accepts X-Dbsc-Session-Id for this demo.
